# CM311 HMI HDMI 1280x720 Timing

Final observed-good display mode for the 1280x720 CM311 HMI image:

- EDID firmware: `cm311_1280x720_v723_728_756_nhsync_nvsync.bin`
- DRM mode: `1280x720 59.22 Hz`
- Pixel clock: `74.500 MHz`
- Timing: `1280 1344 1472 1664 / 720 723 728 756`
- Sync polarity: `-hsync -vsync`
- HDMI state: `max_requested_bpc=8`, `output_format=RGB`, `is_limited_range=n`

The image overlay installs the EDID files under:

```text
/usr/lib/firmware/edid/
```

The boot command line selects the final EDID:

```text
video=HDMI-A-1:D video=HDMI-1:D drm.edid_firmware=HDMI-A-1:edid/cm311_1280x720_v723_728_756_nhsync_nvsync.bin
```

`azs-hmi-cm311.service` also gets an `ExecStartPre` helper that applies:

```text
modetest -M meson -w "35:max bpc:8"
```

Previous stable and rollback EDIDs kept in the image:

```text
cm311_1280x720_left60_1344_1472_1624_723_728_748.bin
cm311_1280x720_left_1344_1472_1624_723_728_748.bin
```

To roll back on a running panel, change only `drm.edid_firmware` in
`/boot/uEnv.txt` to the rollback file, keep the same `video=...@60D` and
`max bpc=8`, then reboot.

Helper commands on the panel:

```sh
cm311-select-hdmi-edid final
cm311-select-hdmi-edid previous
cm311-select-hdmi-edid rollback
cm311-select-hdmi-vtotal final
reboot
```
