LB2002 pinprobe candidate map

Purpose:
- boot with SDIO host disabled
- drive candidate SDIO and auxiliary lines as GPIO from userspace
- probe module pads with an oscilloscope

Candidate lines:
- 64  GPIOX_0   corrected SDIO_D0
- 65  GPIOX_1   corrected SDIO_D1
- 66  GPIOX_2   corrected SDIO_D2
- 67  GPIOX_3   corrected SDIO_D3
- 68  GPIOX_4   corrected SDIO_CLK
- 69  GPIOX_5   corrected SDIO_CMD
- 70  GPIOX_6   corrected CHIP_EN
- 81  GPIOX_17  corrected RST_N
- 82  GPIOX_18  corrected INT

How to use:
- single pin:
  /usr/local/sbin/lb2002-pinprobe-wave.sh 68 250

- sweep all candidates:
  /usr/local/sbin/lb2002-pinprobe-sweep.sh 8 250

Interpretation:
- current candidate map assumes the earlier oscilloscope sweep was shifted by +1 line
- if oscilloscope confirms the expected meander on a module contact, the physical line mapping is confirmed
- if no meander appears on expected SDIO contacts, the physical layer / pin mapping hypothesis remains open
